# Self-Reflection Skill: Design Review

## 概述

self-reflection 是一個後設認知 skill，讓 OpenCode agent 能從自身經驗學習：
工具報錯時分析原因、任務成功時提取模式、Session 結束時總結學習，
並根據學習內容自動產生新的 skill 存入 `.agents/skills/` 或 `.opencode/skills/`。

## 設計目標

1. **自動反省** — 無需使用者介入，agent 自動從錯誤中學習
2. **模式提取** — 從成功的解決方案中提取可重用模式
3. **技能產生** — 將學習成果轉化為結構化的 SKILL.md
4. **持續改善** — 形成「使用→反省→學習→改善」的正向迴圈

## 審查歷程（8 輪 + 計畫審查）

| 輪次 | 漏洞數 | 主題 | 關鍵發現 |
|------|--------|------|---------|
| 第 1 輪 | 8 | 架構設計 | L1 無法自動觸發、JSON 競態、循環依賴 |
| 第 2 輪 | 9 | 實作細節 | 錯誤分類粒度、低價值判定、token 成本 |
| 第 3 輪 | 10 | 邊界情況 | 引導期 UX、日誌增長、隱私、並行衝突 |
| 第 4 輪 | 8 | 系統整合 | Heartbeat 中斷、日誌損毀、同名衝突 |
| 第 5 輪 | 5 | 深層假設 | 自我服務偏誤、effectiveness 度量 |
| 第 6 輪 | 10 | UX/演化 | 可讀性 vs 完整性矛盾、技能爆炸、收斂 |
| 第 7 輪 | 10 | 根本假設 | 歸因偏誤、domain 知識遺漏、回饋迴圈 |

## 核心設計決策

### 1. 三層反省架構

```
L1: 錯誤即時反省 → 節流後產生 draft
L2: 成功模式提取 → 跨 session 比對後產生 draft  
L3: 定期總結 → 合併/歸檔/收斂判斷
```

### 2. 分層配置系統

```
Layer 1: 內建預設值 (sane defaults)
Layer 2: SKILL.md 前段可配置參數
Layer 3: Per-session 臨時覆寫
```

### 3. 洋蔥結構

```
表層 (~20 行): 緊急時最低可行指令
中層 (~50 行): 標準流程  
深層 (~150 行): 完整邊界規則 (F1-F16)
```

### 4. 防止循環繁衍

- `reflectionDepth` 限制反省深度
- Child skill 標記 `parent: self-reflection`
- Child skill 報錯時更新自身，不產生 meta-skill

## 關鍵指標

- **覆蓋漏洞**: 96 項（全數修復）
- **硬編碼參數**: 17 個（全部 Layer 2 可配置）
- **模糊規則**: 16 個（全部 F1-F16 明確定義）
- **SKILL.md 行數**: 614 行（表層 30 + 中層 130 + 深層 250 + 其他 200）

## 安全性考量

1. 日誌不記錄工具參數內容
2. 所有自動 skill 標記 `generated: auto` + `status: draft`
3. `deleted_skills` 清單防止已刪除 skill 再生
4. `max_reflection_depth` 防止無限遞迴
5. Platform 標記避免跨平台誤用

## 已知限制

1. SKILL.md 為被動文字，無法 100% 強制 agent 執行 — 需依賴強制循環演算法
2. 首次使用需 3 session 引導期才有足夠資料產出
3. 跨 session 比對使用 hash，無法理解語意相似性
